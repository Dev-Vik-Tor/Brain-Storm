## Pre-Deployment Checklist

Complete every item before proceeding. Check off as you go.

### Code & CI

- [ ] All CI checks green on the release commit
  - Backend tests passing
  - Frontend build successful
  - Contract tests passing
  - Linting checks passed
- [ ] Release PR reviewed and merged to `main`
- [ ] GitHub Release created and tagged (e.g., `v1.2.0`)
- [ ] `CHANGELOG.md` updated (auto-generated by release-please)
- [ ] No open `P0`/`P1` bugs against this release
- [ ] Security audit completed (if applicable)

### Database

- [ ] Database backup taken and verified restorable:
  ```bash
  pg_dump -h $DATABASE_HOST -U $DATABASE_USER brain-storm \
    | gzip > backup-$(date +%Y%m%d-%H%M%S).sql.gz
  ```
- [ ] Backup stored in secure location (S3, etc.)
- [ ] Pending migrations reviewed (`apps/backend/src/migrations/`)
- [ ] Migrations tested on staging with production-size dataset
- [ ] Confirm `synchronize: false` in `data-source.ts` (never auto-sync in production)
- [ ] Database connection pooling configured
- [ ] Backup retention policy verified

### Secrets & Environment

- [ ] All required env vars present in production secret store
- [ ] `JWT_SECRET` is at least 64 random bytes
- [ ] `STELLAR_SECRET_KEY` is the correct mainnet issuer keypair
- [ ] Contract IDs in env match `scripts/deployed-contracts.json` for `mainnet`
- [ ] API rate limits configured
- [ ] CORS origins whitelist updated
- [ ] Secrets rotation schedule verified

### Infrastructure

- [ ] Load balancer health checks configured
- [ ] Auto-scaling policies reviewed
- [ ] Monitoring and alerting enabled
- [ ] Log aggregation configured
- [ ] CDN cache invalidation plan ready
- [ ] DNS records verified
- [ ] SSL/TLS certificates valid (not expiring within 30 days)

### Communication

- [ ] Maintenance window scheduled (if needed)
- [ ] Stakeholders notified of deployment time
- [ ] Incident response team on standby
- [ ] Rollback plan reviewed with team

## Deployment Process

### Phase 1: Smart Contracts

Deploy contracts to mainnet (if changes made):

```bash
# 1. Verify contract changes
git diff main~1 contracts/

# 2. Build contracts
cd contracts
cargo build --target wasm32-unknown-unknown --release

# 3. Deploy to mainnet
./scripts/deploy.sh mainnet analytics
./scripts/deploy.sh mainnet token
./scripts/deploy.sh mainnet shared

# 4. Verify deployment
stellar contract invoke \
  --network mainnet \
  --id <CONTRACT_ID> \
  -- get_version

# 5. Update contract IDs in env
# Edit scripts/deployed-contracts.json with new contract IDs
```

**Rollback:** Revert to previous contract version (see [Rollback Procedures](#rollback-procedures))

### Phase 2: Database Migrations

Run pending migrations:

```bash
# 1. Connect to production database
export DATABASE_HOST=prod-db.example.com
export DATABASE_NAME=brain-storm
export DATABASE_USERNAME=postgres

# 2. Verify pending migrations
npm run db:migration:show

# 3. Run migrations
npm run db:migrate

# 4. Verify migration success
npm run db:migration:show  # All should show "success"

# 5. Verify data integrity
npm run db:verify-integrity
```

**Rollback:** Revert migrations (see [Rollback Procedures](#rollback-procedures))

### Phase 3: Backend Deployment

Deploy NestJS API:

```bash
# 1. Build backend
npm run build:backend

# 2. Run tests
npm run test:backend

# 3. Deploy to production
# Option A: Docker
docker build -f apps/backend/Dockerfile -t brain-storm-api:v1.2.0 .
docker push brain-storm-api:v1.2.0

# Option B: Kubernetes
kubectl set image deployment/brain-storm-api \
  brain-storm-api=brain-storm-api:v1.2.0 \
  --namespace=production

# 4. Monitor rollout
kubectl rollout status deployment/brain-storm-api -n production

# 5. Verify API health
curl https://api.brainstorm.dev/health
```

**Expected response:**
```json
{
  "status": "ok",
  "timestamp": "2026-05-29T15:42:11Z",
  "version": "1.2.0"
}
```

### Phase 4: Frontend Deployment

Deploy Next.js app:

```bash
# 1. Build frontend
npm run build:frontend

# 2. Run tests
npm run test:frontend

# 3. Deploy to CDN/hosting
# Option A: Vercel
vercel deploy --prod

# Option B: AWS S3 + CloudFront
aws s3 sync apps/frontend/.next/static s3://brain-storm-cdn/
aws cloudfront create-invalidation --distribution-id E123ABC --paths "/*"

# 4. Verify deployment
curl https://brainstorm.dev/health
```

**Expected response:** 200 OK with app shell

### Phase 5: Health Checks

Verify all systems operational:

```bash
# 1. API health
curl https://api.brainstorm.dev/health

# 2. Database connectivity
curl https://api.brainstorm.dev/v1/health/db

# 3. Stellar network connectivity
curl https://api.brainstorm.dev/v1/health/stellar

# 4. Frontend accessibility
curl https://brainstorm.dev

# 5. Check logs for errors
kubectl logs -f deployment/brain-storm-api -n production --tail=100
```

## Rollback Procedures

### Rollback Backend

```bash
# 1. Identify previous stable version
git tag | grep v | sort -V | tail -5

# 2. Rollback deployment
kubectl rollout undo deployment/brain-storm-api -n production

# 3. Verify rollback
kubectl rollout status deployment/brain-storm-api -n production

# 4. Verify API health
curl https://api.brainstorm.dev/health

# 5. Check logs
kubectl logs -f deployment/brain-storm-api -n production --tail=50
```

### Rollback Frontend

```bash
# 1. Identify previous stable version
git tag | grep v | sort -V | tail -5

# 2. Rollback CDN
aws cloudfront create-invalidation --distribution-id E123ABC --paths "/*"

# 3. Redeploy previous version
git checkout v1.1.0
npm run build:frontend
vercel deploy --prod

# 4. Verify frontend
curl https://brainstorm.dev
```

### Rollback Database

```bash
# 1. Stop application
kubectl scale deployment/brain-storm-api --replicas=0 -n production

# 2. Restore from backup
pg_restore -h $DATABASE_HOST -U $DATABASE_USER -d brain-storm \
  backup-20260529-154211.sql.gz

# 3. Verify data integrity
npm run db:verify-integrity

# 4. Restart application
kubectl scale deployment/brain-storm-api --replicas=3 -n production

# 5. Verify health
curl https://api.brainstorm.dev/health
```

### Rollback Smart Contracts

```bash
# 1. Identify previous contract version
stellar contract info --network mainnet --id <CONTRACT_ID>

# 2. Deploy previous version
git checkout v1.1.0
cd contracts
cargo build --target wasm32-unknown-unknown --release
./scripts/deploy.sh mainnet analytics

# 3. Update contract ID in env
# Edit scripts/deployed-contracts.json

# 4. Verify contract
stellar contract invoke \
  --network mainnet \
  --id <CONTRACT_ID> \
  -- get_version
```

```bash

echo "🔍 Verifying deployment..."

# 1. API health
echo "✓ Checking API health..."
curl -f https://api.brainstorm.dev/health || exit 1

# 2. Database connectivity
echo "✓ Checking database..."
curl -f https://api.brainstorm.dev/v1/health/db || exit 1

# 3. Stellar connectivity
echo "✓ Checking Stellar network..."
curl -f https://api.brainstorm.dev/v1/health/stellar || exit 1

# 4. Frontend accessibility
echo "✓ Checking frontend..."
curl -f https://brainstorm.dev | grep -q "Brain-Storm" || exit 1

# 5. API endpoints
echo "✓ Testing API endpoints..."
curl -f https://api.brainstorm.dev/v1/courses || exit 1
curl -f https://api.brainstorm.dev/v1/users/me -H "Authorization: Bearer $TEST_TOKEN" || exit 1

# 6. Database queries
echo "✓ Verifying database integrity..."
npm run db:verify-integrity || exit 1

echo "✅ All verification checks passed!"
```

## Deployment Troubleshooting

### Issue: API fails to start

**Symptoms:** Pod crashes, logs show connection errors

**Diagnosis:**
```bash
kubectl logs deployment/brain-storm-api -n production --tail=100
```

**Solutions:**
1. Verify environment variables: `kubectl get configmap brain-storm-config -o yaml`
2. Check database connectivity: `psql -h $DATABASE_HOST -U $DATABASE_USER -d brain-storm`
3. Verify secrets: `kubectl get secret brain-storm-secrets -o yaml`
4. Rollback to previous version: `kubectl rollout undo deployment/brain-storm-api`

### Issue: Database migration fails

**Symptoms:** Migration hangs or errors

**Diagnosis:**
```bash
npm run db:migration:show
psql -h $DATABASE_HOST -U $DATABASE_USER -d brain-storm -c "SELECT * FROM migrations;"
```

**Solutions:**
1. Check for locks: `SELECT * FROM pg_locks;`
2. Kill blocking queries: `SELECT pg_terminate_backend(pid) FROM pg_stat_activity WHERE state = 'active';`
3. Rollback migration: `npm run db:migration:revert`
4. Review migration code for issues

### Issue: High error rate after deployment

**Symptoms:** 5xx errors, timeouts

**Diagnosis:**
```bash
kubectl logs -f deployment/brain-storm-api -n production | grep -i error
kubectl top pods -n production
```

**Solutions:**
1. Check resource usage (CPU, memory)
2. Scale up replicas: `kubectl scale deployment/brain-storm-api --replicas=5 -n production`
3. Check database connection pool: `SELECT count(*) FROM pg_stat_activity;`
4. Rollback deployment: `kubectl rollout undo deployment/brain-storm-api`

### Issue: Frontend not loading

**Symptoms:** Blank page, 404 errors

**Diagnosis:**
```bash
curl -v https://brainstorm.dev
aws cloudfront get-distribution-config --id E123ABC
```

**Solutions:**
1. Invalidate CDN cache: `aws cloudfront create-invalidation --distribution-id E123ABC --paths "/*"`
2. Verify S3 bucket permissions
3. Check DNS records: `nslookup brainstorm.dev`
4. Rollback frontend: `git checkout v1.1.0 && npm run build:frontend && vercel deploy --prod`

## Deployment Templates

### Kubernetes Deployment Template

```yaml
# k8s/deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: brain-storm-api
  namespace: production
spec:
  replicas: 3
  selector:
    matchLabels:
      app: brain-storm-api
  template:
    metadata:
      labels:
        app: brain-storm-api
    spec:
      containers:
      - name: brain-storm-api
        image: brain-storm-api:v1.2.0
        ports:
        - containerPort: 3000
        env:
        - name: DATABASE_HOST
          valueFrom:
            configMapKeyRef:
              name: brain-storm-config
              key: database-host
        - name: JWT_SECRET
          valueFrom:
            secretKeyRef:
              name: brain-storm-secrets
              key: jwt-secret
        livenessProbe:
          httpGet:
            path: /health
            port: 3000
          initialDelaySeconds: 30
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /health
            port: 3000
          initialDelaySeconds: 10
          periodSeconds: 5
        resources:
          requests:
            memory: "256Mi"
            cpu: "250m"
          limits:
            memory: "512Mi"
            cpu: "500m"
```

### Docker Deployment Template

```dockerfile
# apps/backend/Dockerfile
FROM node:18-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build:backend

FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY --from=builder /app/apps/backend/dist ./dist
EXPOSE 3000
CMD ["node", "dist/main.js"]
```

### GitHub Actions Deployment Template

```yaml
# .github/workflows/deploy-production.yml
name: Deploy to Production

on:
  push:
    tags:
      - 'v*'

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Build backend
        run: npm run build:backend
      
      - name: Build frontend
        run: npm run build:frontend
      
      - name: Push Docker image
        run: |
          docker build -f apps/backend/Dockerfile -t brain-storm-api:${{ github.ref_name }} .
          docker push brain-storm-api:${{ github.ref_name }}
      
      - name: Deploy to Kubernetes
        run: |
          kubectl set image deployment/brain-storm-api \
            brain-storm-api=brain-storm-api:${{ github.ref_name }} \
            --namespace=production
      
      - name: Verify deployment
        run: bash scripts/verify-deployment.sh
```
